## 학습 진행 방식 (상세)

사용자가 공부를 시작하면 **이 순서대로** 진행할 것:

### 1단계: 학습 목표 제시
- 이 주제에서 배울 핵심 내용 소개
- 끝나면 뭘 할 수 있게 되는지 설명
- **예지보전에서 어떻게 쓰이는지** 언급

### 2단계: 동기부여
- 왜 이게 필요한지, 실제로 어디에 쓰이는지 설명
- **공장설비 예시** 적극 활용

### 3단계: 핵심 개념 설명
- 이론을 쉽게 풀어서 설명
- 필요하면 그림이나 다이어그램 (ASCII art) 활용
- 코드는 하나씩 차근차근

### 4단계: 실습 예제
- 직접 따라할 수 있는 예제 제공
- **센서 데이터, 설비 로그 등 실무 데이터 예시 활용**
- 결과가 어떻게 나오는지 미리 알려주기

### 5단계: 스스로 해보기
- 연습 문제 2-3개 제시
- 힌트는 요청하면 제공
- 정답은 사용자가 시도한 후에 확인

### 6단계: 퀴즈
- 이해도 확인을 위한 간단한 퀴즈 2-3개
- 개념 확인 + 실전 문제 섞어서

### 7단계: 정리
- 오늘 배운 핵심 개념/코드 요약
- 다음 주제 예고

## 복습 시스템 (3단계 진행)

```
review/
├── 1-new/terms.md       ← 새로 배운 용어/개념
├── 2-practicing/terms.md ← 복습 중 (1차 퀴즈 통과)
└── 3-mastered/terms.md   ← 완벽 암기 (2차 퀴즈 통과)
```

### 복습 프로세스

1. **매 수업 시작 시**: `1-new/terms.md`와 `2-practicing/terms.md`에서 랜덤 퀴즈 출제
2. **퀴즈 결과에 따라 승급**:
   - 2-practicing에서 정답 → `3-mastered`로 이동
   - 오답 → 현재 단계에 유지
3. **챕터 학습 후**: 틀린 것/몰랐던 것을 `2-practicing`에 추가
4. **새로 배운 개념**: `1-new`에 추가 → 다음 세션 퀴즈에서 통과하면 `2-practicing`으로

## 노트 작성 가이드 (튜토리얼 스타일)

notes/ 폴더의 노트는 **"이걸 보면 남에게 설명할 수 있다"** 수준으로 작성할 것.

### 포함할 내용
- **핵심 개념**: 이론 정리 (왜 필요한지, 뭔지)
- **다이어그램**: ASCII art나 표로 흐름/구조 시각화
- **코드/함수 정리표**: 한눈에 보는 요약
- **실습 가이드**: 따라하기 좋은 순서 (복붙 가능하게)
- **주의사항 & 흔한 실수**: 학습 중 겪은 실수, 헷갈리는 포인트

### 작성 원칙
- 기존 notes/ 구조에 맞춰 저장
- 개념 설명 → 실습 → 주의사항 순서로 구성
- 실습 예제는 **처음부터 끝까지 따라할 수 있게** 작성

## 학습 완료 후 할 일

- `logs/` 폴더에 학습 일지 기록 (`YYYY-MM-DD.md`)
- `notes/` 폴더에 학습 노트 작성
- `review/` 폴더에 새로 배운 용어/틀린 것 추가
- README.md의 체크리스트 업데이트 ([ ] → [x])
- README.md의 학습 일지 테이블에 기록 추가
- 커밋 메시지 형식: `docs: [주제] 학습 완료 - [주요 키워드]`
- **상위 study/ 폴더는 별도 git repo가 아니므로 커밋할 필요 없음.** 각 과목 폴더의 레포만 커밋 & 푸시하면 됨.

## PDF 읽기 주의사항

PDF를 읽어야 할 때 반드시 지킬 것:

### 전체를 한 번에 읽지 말 것!

PDF가 크면 (5MB 이상, 또는 20페이지 이상) 전체를 한 번에 읽으면 컨텍스트가 꽉 차서 **세션이 먹통**될 수 있음.

### 반드시 페이지별로 나눠서 읽기

```markdown
# 좋은 예
"1-3페이지만 읽어줘"
"특정 섹션만 읽어줘"

# 나쁜 예 (세션 먹통됨 - 절대 금지)
"전체 읽어줘"
```

### 사용자가 PDF 읽기를 요청하면

1. 먼저 몇 페이지인지 확인
2. 20페이지 이상이면 **반드시** 섹션별로 나눠서 읽자고 제안
3. 1-3페이지씩 나눠서 진행
