# NumPy 기초

## 왜 NumPy인가?

- Python 리스트보다 **10~100배 빠름** (C로 구현, 메모리 연속 배치)
- Pandas, Scikit-learn, PyTorch 등 거의 모든 데이터/ML 라이브러리의 **기반**
- 예지보전: 센서 데이터(진동, 온도, 압력)가 초당 수천 개 → 빠른 처리 필수

---

## 1. ndarray — 핵심 자료구조

```
Python 리스트              NumPy ndarray
┌───┬───┬───┬───┐         ┌───┬───┬───┬───┐
│72 │"a"│3.0│True│         │72 │73 │71 │74 │ ← 같은 타입만, 메모리 연속
└───┴───┴───┴───┘         └───┴───┴───┴───┘
다양한 타입 가능 / 느림     같은 타입 / 빠름
```

### 배열 만들기

```python
import numpy as np

# 직접 생성
arr = np.array([72.1, 73.5, 71.8])

# 2차원 (센서 3개 × 시간 4개)
data = np.array([
    [72.1, 73.5, 71.8, 74.2],  # 센서1
    [35.2, 36.1, 34.8, 35.9],  # 센서2
    [1.02, 1.05, 1.01, 1.03],  # 센서3
])
data.shape  # (3, 4)
data.dtype  # float64
```

### 배열 생성 함수

| 함수 | 설명 | 예시 |
|------|------|------|
| `np.zeros((m,n))` | 모두 0 | `np.zeros((3,4))` |
| `np.ones((m,n))` | 모두 1 | `np.ones((2,3))` |
| `np.full((m,n), v)` | 모두 v | `np.full((2,3), 7.0)` |
| `np.arange(s,e,step)` | 범위 (**끝값 미포함**) | `np.arange(0,101,5)` |
| `np.linspace(s,e,n)` | 균등 분할 (끝값 포함) | `np.linspace(0,1,5)` |
| `np.eye(n)` | 단위행렬 | `np.eye(3)` |
| `np.random.randn(m,n)` | 표준정규분포 난수 | `np.random.randn(3,4)` |

## 2. 인덱싱 & 슬라이싱

```python
arr = np.array([72.1, 73.5, 71.8, 74.2, 70.5])
arr[0]      # 72.1
arr[-1]     # 70.5
arr[1:3]    # [73.5, 71.8]

# 2차원
data[0, 2]   # 0행 2열
data[:, 1]   # 모든 행의 1열
data[0, :]   # 0행 전체
```

### 불린 인덱싱 (조건 필터링)

```python
temps = np.array([72.1, 85.3, 71.8, 90.2, 70.5])

temps[temps > 80]                    # [85.3, 90.2]
temps[(temps > 80) & (temps < 95)]   # 조건 조합
# ⚠️ and 아니라 & 사용! 각 조건에 괄호 필수!
```

## 3. 벡터 연산 (Vectorization)

반복문 없이 배열 전체에 연산 적용:

```python
temps_f = np.array([72.1, 73.5, 71.8])
temps_c = (temps_f - 32) * 5/9   # 한 줄로 변환

voltage = np.array([220, 221, 219])
current = np.array([5.1, 5.3, 5.0])
power = voltage * current   # 원소별 곱셈
```

## 4. 브로드캐스팅

크기가 다른 배열 간 연산을 자동으로 맞춰줌:

```python
data = np.array([[72, 73, 71, 74],
                 [35, 36, 34, 35]])

means = data.mean(axis=1, keepdims=True)  # (2,1)
deviations = data - means   # (2,4) - (2,1) → 자동 확장!
```

## 5. axis 개념

```
         열 방향 → (axis=1)
       ┌──┬──┬──┬──┬──┐
행     │70│72│71│73│74│ → axis=1: 행을 따라 가로로 연산 (센서별 평균)
방향 ↓ │68│69│70│71│72│
(axis=0)│75│76│74│77│78│
       └──┴──┴──┴──┴──┘
         ↓ axis=0: 열을 따라 세로로 연산 (시간대별 비교)
```

- `axis=0`: 행 방향으로 축소 (세로, 결과는 열 개수만큼)
- `axis=1`: 열 방향으로 축소 (가로, 결과는 행 개수만큼)

## 6. 유용한 함수

|------|------|
| `.mean()` | 평균 |

## 주의사항 & 흔한 실수

- **불린 인덱싱**: `arr[arr > 5]`처럼 배열 이름 그대로 사용 (별도 변수 X)
- **& vs and**: NumPy 조건 조합은 `&`, `|` 사용. `and`, `or` 쓰면 에러
- **괄호 필수**: `(arr > 5) & (arr < 10)` — 괄호 없으면 연산 우선순위 에러
- **arange 끝값**: `np.arange(0, 100, 5)`는 100 미포함. 100까지 원하면 `101`
- **axis 헷갈림**: axis=0은 "0번 축(행)을 따라" → 세로 방향, axis=1은 가로 방향
