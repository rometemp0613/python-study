# 새로 배운 용어/개념

## 01. 왜 테스트가 필요한가

- **테스트 피라미드 (Test Pyramid)**: Unit(많이) → Integration(적당히) → E2E(핵심만) 순서로 테스트를 구성하는 전략. Unit이 기반이 되어야 상위 테스트도 의미가 있음
- **경계값 테스트 (Boundary Testing)**: 조건 분기의 경계에서 값을 테스트하는 기법. 예: 4.49 vs 4.5로 정상/주의 분류 검증
- **테스트 = 행동 명세 (Test as Specification)**: 테스트는 버그 잡는 도구를 넘어, 코드가 어떻게 동작해야 하는지의 실행 가능한 문서. AI 시대에 코드 복제가 쉬워지면서 테스트가 진짜 경쟁력(해자)이 됨
- **리팩토링 안전망 (Refactoring Safety Net)**: 구현 방식을 바꿔도 테스트가 동작을 보장해주는 역할. 예: 순수 Python → NumPy 전환 시 테스트로 결과 동일성 검증
